Web Dev Concepts

Web Servers

When a user wants to visit a webpage they input the Fully Qualified Domain Name (FQDN) into their browser which resolves the IP address using a Domain Name System (DNS) server. The user then connects to the server using the IP address which host the designated homepage. The user's browser loads the designated homepage at the root url which is typically named index with some suffix such as .html, .js, etc. Static content homepages such as this one typically end in .html (hypertext mark up languages) whereas dynmaclly generated homepages end in .js (javascript) or other suffixs.

A users Web Dev/Inspector Networks tab output screen displays the request and response headers which contain valuable information such as stutus code, type, url, last time modified, and method. The reponse body contains the actual data such as the index.html document. If the file is viewed from the web server the url will contain a FQDN such as https://web.engr.oregonstate.edu/~amidonk whereas if it viewed from a local machine the url will be displayed in the following format file:///users/kevin/Desktop/CS%290/M1/a1-amidonk/index.html.

The favicon.ico file has a status of 200 ok because it found on the server however the main.css and main.js have a 404 error meaning the files were not found because they were not found.

A url can consist of a scheme, subdomain, host domain and resources. The scheme is the protocol such as http and https. The subdomain is an optional prefix to the domain name such as www. The host domain is the main address of the website which includes a domain name and a Top Level Domain (TLD). Resources provides the specific location where a requested item can be found.
